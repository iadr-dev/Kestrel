## Which frontend models are NOT working вЖТ REMOVED

- **`microsoft/phi-4-multimodal-instruct`** вАФ NVIDIA function is DEGRADED (500/400
  "DEGRADED function cannot be invoked"). Broken upstream. **Removed** from backend +
  frontend.
- **`qwen/qwen3.5-397b-a17b`** вАФ the NVIDIA-hosted 397B endpoint did not respond within
  240s across repeated runs. Times out; not usable interactively. **Removed**.

### Not broken (corrected)

- **`meta/llama-4-maverick-17b-128e-instruct`** вАФ it's a **vision** model; its job is
  image analysis, not driving the tool loop. Tested with a real image (a photo of Jensen
  Huang) вЖТ described it accurately (NVIDIA CEO, leather jacket, green glow). It is the
  free-tier `$0` vision model (`FREE_VISION_MODEL`). **Kept & re-enabled.** (My initial
  "no tools" verdict measured the wrong thing for a vision model.)
- **`openrouter/free` "didn't call a tool"** вАФ NOT a code bug. Tools are passed
  identically to every model (skill/intent-driven, never gated by model name). On a soft
  prompt it chose to reason-and-answer; given an explicit "еЛЩењЕеСЉеПЂеЈ•еЕЈ" prompt it called
  `get_realtime_quote` correctly. Model decision, not a defect.
- **`gemini-3.1-pro-preview` / `gemini-3.5-flash`** вАФ 429 quota on the current key's free
  tier (3.1 Pro: quota 0; 3.5 Flash: 20/day). A key/plan limit, not a code or model
  defect. Left in the selector.

The four Anthropic models are the only ones that stream **live reasoning** (the

## Tool process вАФ always shown вЬЕ

Per the requirement "tool process must always show when calling tools": on every model
that actually called a tool (Claude √Ч4, GPT √Ч4, Gemini 2.5, all Free/Other except the
broken ones), the `tool_start вЖТ tool_done` progression streamed live with summary +
duration вАФ the frontend `ThinkingTimeline` renders it as the Hermes-CLI-style
`preparingвА¶ вЖТ done Xs` steps. This path was already correct and needed no change.

## Fixes applied

1. **Claude extended thinking enabled** (`router.py` `_stream_claude`): request
   `thinking={"type":"enabled", budget 2048}` for `claude-(opus|sonnet|haiku)-4*`, drop
   `temperature` (required off with thinking), bump `max_tokens` above the budget.
   Capture the streamed `signature_delta` and replay the signed thinking block in the
   assistant turn (`loop.py`) so the tool-use loop doesn't 400. вЖТ live reasoning now
   streams on all Claude models (verified: opus 3 events, haiku 12 events, coexisting
   with tool calls).
2. **OpenAI-compat reasoning surfaced** (`router.py` `_stream_openai`): emit
   `delta.reasoning_content` / `delta.reasoning` as `thinking_delta`. вЖТ `openrouter/free`
   now streams 1392 reasoning events; DeepSeek-R/OpenRouter `:thinking` variants covered.

Non-reasoning models correctly show no thinking timeline (by design) but always show the
tool process.
